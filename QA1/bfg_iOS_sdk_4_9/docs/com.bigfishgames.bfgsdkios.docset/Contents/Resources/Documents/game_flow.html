The Big Fish iOS SDK and iOS Game Flow on Launch

The following illustrations highlight how the SDK affects a game's UI on launch.

Springboard

Game Icon:

  • Big Fish will provide a Bundle Display Name to be shown under your game's icon on the device springboard and in searches.
  • Icons in all sizes will be provided by Big Fish.
  • Initializes bfgManager.
  • On first launch, branding screens will display, followed by Main Menu.
    • On first launch, the Game Discovery Network (GDN) is not connected.
    • If Wifi is on, the GDN will display on the second and subsequent launches.
    • If Wifi is off, the GDN will not display until Wifi is on at the second or subsequent launches.
  • On subsequent launches, will resume games. A resumed game on the Game Discovery Network (GDN) has Resume and Main Menu buttons.
  • Loads the GDN for Freemium games. The GDN is not required for Free-to-Play games, so you need to make sure "dashboard_active" is set to 0.
  • The user may not see the GDN depending on the bfg_default_settings.json file settings downloaded from the server. These will overwrite your local settings.
Note
Downloaded server settings may change your local settings.

For more information, see Freemium Usability Requirements and bfgManager.

Calls:

  • [bfgManager initializeWithViewController:myRootViewController] (first time)

The following helper methods are available for more information:

  • [bfgManager isFirstTime]
  • [bfgManager isInitialized]
  • [bfgManager isInitialLaunch]
  • [bfgManager sessionCount]

Default blue watermark

Watermarks are default images that are provided by your producer.

  • iPad: This default background is a customized blue watermark specific to your game.
  • iPhone: The default background is plain blue. It is dynamically generated, and no image is provided.

Game Discovery Network

Note
This architecture also applies to all types of games. For example, a Free-to-Play game would have "dashboard_active" set to 0, and so the GDN would not display.

The first time a game launches, it reads the settings in the bfg_default_settings.json file to determine whether or not to show the GDN.

On subsequent launches, if the game is able to connect to the server, it reads the settings from the server.

Any time the user connects to the server, default settings may be altered. For example, after the game successfully connects to the server once, the bfg_default_settings.json file may no longer be used.

Calls:

  • [bfgPurchase startService] – Initiates the bfgPurchase process. Start the service when a game is ready to handle a purchase, as soon as possible after launch.
  • [bfgPurchase acquireProductInformation] – For each product offered for purchase in the game. Start right after [bfgPurchase startService]. It runs in the background, and updates current product information.
Note
After a user pauses the game, the dashboard enters a Pause/Resume mode. No additional code is needed to create or initialize this view, and display of this view will be handled automatically by the bfgManager. For more information, see Game Discovery Network.

Big Fish branding

The Big Fish Game logo displays for all games. Use the most recent “splash screen“ image from Big Fish. This image contains the Big Fish logo and no other text, so that the branding does not need to be localized.

It is located in the bfgbranding_resources.zip file.

  • Add your own branding assets to the asset bundle.
  • In the branding.json file in the bfglib_gamespecific_resources directory, specify the assets to display for each supported device and orientation. Assets will be displayed for each device in the order listed in this file.

  • If file size is a consideration, you can use WebP to compress your images significantly. See WebP - A new image format for the Web for more information on WebP. The SDK will support this image type automatically, and already includes the Big Fish branding in this format.

Calls:

  • [bfgManager startBranding]

Developer branding splash screen

Optional.

The bfgbranding_resources.zip file contains the branding.json file that the bfgBrandingViewController class uses for displaying branding images. You can include your own branding images by adding your file names to branding.json.

Calls:

  • [bfgManager startBranding]

Game loading

You will receive a notification that the branding is complete (BFGBRANDING_NOTIFICATION_COMPLETED), and the game loads.

Create new profile

Optional.

Used for creating a new game profile. Displays over greyed Main Menu.

Main Menu

The appearance of the Main Menu depends on whether the user is playing for the first time or returning to the game, or has purchased the full game ("unlock").

For more information, see Freemium Main Menu Best Practices, bfgManager, bfgRating, and bfgSplash.

Calls:

  • [bfgManager startAds];
  • [bfgManager adsRunning];
  • [bfgManager showMoreGames];
  • [bfgRating canShowMainMenuRateButton];
  • [bfgRating hasRatedApp];
  • [bfgRating mainMenuRateApp];
  • [bfgSplash displayNewsletter:baseController];
