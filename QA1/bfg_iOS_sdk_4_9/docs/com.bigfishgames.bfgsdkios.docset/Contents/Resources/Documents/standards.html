Information about coding standards, tools, and device support.

Big Fish has established a set of requirements and recommendations for developers to use when they produce games for distribution to the Apple App Store. Games must meet all requirements before they can be accepted and distributed.

  • See the Freemium Development Checklist for a summary of the most important Freemium requirements.
  • See the Freemium Usability Requirements for more information about Freemium user experience requirements.
  • See the Free-to-Play Development Checklist for a summary of the most important Free-to-Play requirements.
  • See the Free-to-Play Usability Requirements for more information about Free-to-Play user experience requirements.

Development Tools

Use the following development tools and libraries to create applications for Big Fish:

  • Xcode: Xcode is the exclusive development platform for Big Fish iOS: Xcode versions 5.0 and up (Apple requirement as of Feb. 1, 2014), using .xcodeproj files as opposed to Xcode workspaces.
  • SVN: Big Fish provides an external SVN repository for build submissions.
  • JIRA: Big Fish uses JIRA to track bugs. You must update the bug database before you submit changes for any build.
  • Big Fish iOS Software Development Kit (Big Fish iOS SDK): The SDK contains the code to implement the game features that are unique to Big Fish.
Note
If you use additional development tools and libraries, you must include them in build submissions to Big Fish.

Big Fish iOS Software Development Kit (Big Fish iOS SDK)

The Big Fish iOS SDK contains a framework for the major Big Fish components that you can quickly add to your game project. These components include:

  • Ad integration
  • Promotional features, such as the Game Discovery Network (GDN) and the iSplash! newsletter.
  • Branding
  • Rating
  • Reporting
  • In-game purchase
  • Login

The Big Fish iOS SDK also includes an example code project and documentation.

Big Fish adds an external link from the trunk branch of the external SVN repository for your title, to the current version of the Big Fish iOS SDK. When you check out code from the trunk branch of your repository, the build team adds a folder containing the latest version of the Big Fish iOS SDK.

  • For more information about the framework, see Classes and Components.

Third-party tools and libraries

Third-party tools and libraries that you use to build your game project must meet the following requirements:

  • Submit tools with the build: You must submit all third-party tools required for a build with the build. Big Fish will not accept links to tools or requests to download a tool or library.
  • Compatible with Mac OS: All tools that you use to build a game target must run on Mac OS. Big Fish will not accept builds that use tools that require Microsoft Windows.
Important: Big Fish will reject builds that do not fulfill the above requirements. Rejected builds are not eligible for milestone payments.
  • Scripts: If a tool is required to build your project in Xcode, include a script that runs the tool. Do not include scripts that add language folders.
  • Installation instructions: If a tool requires installation, say so in your build notes and include installation instructions. Documentation provided along with the tool will suffice. If possible, provide a script to install the tool. If a tool changes during production, you must document the changes in your build notes.
  • Pre-compiled source code: The complete pre-compiled source code must be included in your project.
  • Open standards: All libraries must comply with open standards, and support a free license, unless Big Fish already has a paid license.
  • Library source code: All libraries must be included as source, unless Big Fish approves an exception.
  • Add libraries to the project: All libraries must be included as part of the Xcode project, either as a separate target, or directly within the game project.
  • No converted data: Do not provide executables or encrypted data.
  • No private APIs: Apple does not allow the use of private APIs. Do not use private APIs in your code.
  • Apple requirements: All libraries must comply with all of Appleâ€™s requirements for third-party libraries.

Coding Standards

To facilitate game development among multiple developers and other people in the production process who are not primarily developers, you must use consistent coding standards.

The Big Fish producers have created a set of recommended coding standards. These standards apply to languages derived from C, including Objective C, C, and C++.

Most developers use one of these languages. If you use another language, these standards might not apply.

  • Use spaces, not tabs, for indentation. Use four (4) spaces for each level of indentation.
  • Place braces on their own lines.
  • Use braces in case statements, and add one break statement after the case statement.
  • Limit each line to one statement.

If you use an existing set of consistent coding standards that vary from these recommendations, Big Fish does not require that you change your standards to match the recommendations. However, Big Fish does require four (4) spaces for each level of indentation.

Device and Software Support

Big Fish requires all new games to run on iOS 5.0 or later versions and to support the armv7 architecture.

The game must detect the device type automatically and provide features based on the detected device type.
