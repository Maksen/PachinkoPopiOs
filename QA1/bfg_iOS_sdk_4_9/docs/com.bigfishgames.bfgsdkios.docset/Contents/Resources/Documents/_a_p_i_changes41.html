4.1 API Changes

Changes to API between 4.0 and 4.1.

The primary focus to the changes below was to simplify entrance into singleton instances by enforcing the use of class methods. Note that in many cases the only change is removing an instance method (demarcated with a '-') with a class method (demarcated with a '+').

bfgutils.h
Added two macros for creating creating singletons in a thread-safe manner:
#define SHARED_INSTANCE_IMPL(CLASS_NAME)
#define SHARED_INSTANCE_IMPL_SPECIAL(CLASS_NAME, INIT_METHOD)
Both create a class method sharedInstance implementation and a static named _sharedInstance. For example:
@interface Foo
+ (Foo *)sharedInstance;
@end
@implementation Foo
SHARED_INSTANCE_IMPL(Foo);
@end
If you have a special init method to call you could do this:
@interface Foo
{ CGRect myRect; }
+ (Foo *)sharedInstance;
@end
@implementation Foo
- initWithRect:(CGRect)rect { ... }
SHARED_INSTANCE_IMPL(Foo, initWithRect:CGRectMake(10,10,10,10));
@end
