Best practices for implementing the Buy Wall in your Freemium games.

Creating an Effective Buy Wall

Buy Wall best practices are critically important to a game. An effective Buy Wall maximizes the opportunity to convert a "free" user into a paying customer, and can be the difference between a good game and a great game’s financial performance.

Note
  • Buy Wall best development practices generally apply to all mobile platforms, but requirements vary by platform.
  • The examples shown assume the game is played in landscape orientation, but the same general best practices would apply for portrait-oriented games as well.

Here is a mock-up of a Buy Wall containing all required components, which are discussed in more detail below.

The Unlock Now! button on the Buy Wall

Tapping on the Unlock Now! button from the Main Menu displays a full page upsell, or Buy Wall, that prompts the user to purchase the full game content. The Buy Wall page should be attractive and include a single large and prominently placed Unlock Now! button.

Note
The Unlock Now! button is the only button presented on the Buy Wall.

Unlock Now! button location

The Unlock Now! button can be in the center, lower center, right center, or lower right corner, but avoid placing the button anywhere near the left or the top of the Buy Wall.

The Unlock Now! button should be located somewhere within the green area of the Buy Wall shown below.

Unlock Now! button hit box

The hit box for the Unlock Now! button should be generous. If the user gets remotely close to the button, it should invoke the purchase call.

Unlock Now! button formatting

The Unlock Now! button should stand out from the rest of the Buy Wall. The button should have sparkles, or glowing text, or movement, or anything else that makes the button stand out.

The button should be the very first thing users notice when they see the Buy Wall for the first time.

Selling the game

The Buy Wall should include compelling screenshots and art assets from future parts of the game, along with brief copy that compels the user to purchase the game. Make sure the purpose of the Buy Wall is completely clear to the user.

  • If it is a Collector's Edition game, it must include all of the Bonus content offered.
  • Interactivity with the content is encouraged. For example:
    • Clicking on a thumbnail image and seeing it take over the Buy Wall screen
    • Rotating images
    • Compelling cut scenes
  • Keep Buy Wall graphic design simple, bold, and compelling to sell users visually on why they should unlock the game to continue the adventure. Too much activity on the Buy Wall may stop the user from going on.
  • To convey the purpose of the Unlock Now! button, header text may appear beneath the game image, that says "Unlock the full version to continue your adventure!" or a similar message. If you are unsure of the best header text to use, contact your Big Fish Producer.

Tappable text: Already purchased?

At the bottom left, include small tappable text in a plain font entitled Already Purchased?.

Note
This text should not be a button and should not draw focus away from the Unlock Now! button.

The "Already purchased?" text should be located somewhere in the green area shown below.

The Already Purchased? text should prompt a restoreCompletedTransactions call followed by a prompt for the user to login to the App Store.

If the Full Content Unlock purchase has already been completed, a system message will notify the user. For more information, see See Making Freemium In-App Purchases.

Close control functionality

The user can return to the Main Menu by clicking on an X close control in the upper right hand corner of the Buy Wall. The hit box for the X can definitely be bigger than the X represented – do not trap users who want to exit the Buy Wall.

The X close control for the Buy Wall should be located in the green area shown below.

Buy Wall timing

The selection of the Buy Wall moment is very important and will vary from game to game.

As a general rule, the Buy Wall should be invoked at the first compelling cliffhanger moment that happens, approximately 25 to 45 minutes into gameplay. You cannot wait for an hour for a compelling cliffhanger moment.

Presenting the Buy Wall during a mundane dialog or hidden object scene will not work.

Note
A Buy Wall invocation should never take more than 45 minutes.

Session Data

Big Fish tested a randomly selected game with a ~60 minute Buy Wall. Of the devices that downloaded and played the game, 43% of them never made it to the ~60 minute Buy Wall.

Buy Wall In-App Purchase process

When tapped, the Unlock Now! button initiates an In-App Purchase confirmation dialogue. This dialogue is controlled by Apple, with the exception of the In-App Purchase title and price, which is set in iTunes Connect by your Big Fish Producer.

Here is an example from the iPad release of Puppet Show: Mystery of Joyville.

The bfgPurchase class is required for In-App Purchases. For more information, see Making Freemium In-App Purchases.

After purchasing the game, players should see a confirmation message and then be taken directly back to the Main Menu or to the Buy Wall point in the game, so they can immediately continue playing the game.

When the user returns to the Main Menu, the Unlock Now! button should no longer appear. In some cases, it may make sense to replace this button with a new button. Consult your Big Fish Producer to determine if this applies to your game.

iTunes Connect

Big Fish will set up the iTunes Connect store for each app and will provide any information necessary for integration.

Buy Wall examples

The following before and after examples illustrate Buy Wall best practices. The after examples feature small adjustments to an existing Buy Wall to ensure it conforms with best practices.

Lockwood Manor: Mystery of the Ancients Collector’s Edition HD

  • X close control added to upper right.
  • Unlock Now! button prominently displayed, instead of "purchase" message.
  • Already purchased? tappable text added at lower left, instead of button at lower right.
  • "Later" button removed from lower left, making an uncluttered page.

Death at Fairing Point: A Dana Knightstone Novel Collector’s Edition HD

  • X close control added to upper right.
  • Unlock Now! button prominently displayed, instead of a "buy now" or similar message.
  • Already purchased? tappable text added at lower left, instead of "Already Paid?" button at lower right.
  • "Later" button removed from lower right.

Beauty and the Beast: Mystery Legends Collector’s Edition HD

  • Unlock Now! button prominently displayed, instead of "buy now" message.
  • Already purchased? tappable text added at lower left, instead of "Already Purchased" button at lower right.
