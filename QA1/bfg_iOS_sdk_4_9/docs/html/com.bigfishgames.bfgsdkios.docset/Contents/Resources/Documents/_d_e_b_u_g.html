Debugging with the SDK

Step-by-step instructions on how to turn on debugging logs with the Big Fish iOS SDK.

Use this technique if you want to get details about what the SDK is doing quietly in the background. This is helpful if your application is not reporting installs, session events, or purchases.


  • Locate the "Framework Search Paths" in your target's build settings.

  • Open the setting and note the current location for the bfg_iOS_SDK framework.

  • Confirm there is only one listing for the SDK. NOTE: Having more then one path to the SDK can cause unexpected results.

  • Append "/frameworkDebug" to the setting for the SDK's "Framework Search Paths."

  • Run your application.

  • If this is the initial run, you will see a call to device services to get the BFGUDID.

  • Next a call is made to startup to get information from the Big Fish Mobile Services.

  • Here is where the GDN ("dashboard_active") and ads settings ("allowed_ad_types") are updated from the server. If you game is configured by your producer to use Kontagent, this is where you will see those settings.

  • Here is a successful validated purchase logging from our purchase service.

  • At Fast App Switch, a final call to session.php is made to send the session log.
