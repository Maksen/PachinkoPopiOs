Game requirements to qualify for Beta candidacy.

Review Developer Documentation

▢ Reviewed current iOS documentation for current requirements, best practices, and SDK help. 

Device and iOS Support

▢ Tested on the most recent iOS version. 

▢ Backwards compatibility (iOS deployment target) for older iOS versions still supported by Big Fish (5.0 and up). 

▢ Supports all iPhone/iTouch devices 4th Generation and up. 

▢ Supports iPad devices 2nd Generation and up. 

▢ Supports Retina Display on 4th and 5th Generation iPhone/iPod Touch devices. 

▢ Supports Widescreen Display on iPhone 5 devices. No pillarboxing, letterboxing, or decorative borders are used. 

▢ Supports 3rd Generation iPad aspect ratio. (Optional support for iPad Retina Display.) 

▢ Supports Fast App Switching, Pinch-zoom, and Gestures. 

▢ Supports rotation of the device. (Landscape orientation only on iPad devices.) 

▢ Supports any device-specific functions needed for gameplay mechanics (accelerometer, camera, etc.). 

Big Fish iOS SDK

▢ Big Fish iOS SDK was externally linked to the game's EXTSVN repository trunk by Big Fish Mobile Game Ops. 

▢ All required frameworks, all required libraries, and resource bundles (device specific) added to the game project. 

▢ Dashboard initialized (bfgManager), Purchase Service started (bfgPurchase), and all required Big Fish components working; see Classes and Components.

▢ Game specific assets (placeholder icon, developer branding, etc.) included in bfg_gamespecific_resources under the project’s resources directory.

▢ bfg_default_settings.json in bfg_gamespecific_resources updated with game specific details (Product ID and Flurry Key).

▢ bfgpromodashboard_maintitle.json in bfg_gamespecific_resources updated with game specific details (full app title in all languages).

▢ Completed bfgPurchase implementation for In-App Purchases. Custom implementations of In-App Purchase are not accepted; see Making Free-to-Play In-App Purchases.

Main Menu

▢ Main Menu and Review prompt (if required) sent to Big Fish Mobile Producer, all feedback applied, and final draft approved by Big Fish internal review. 

▢ Play, Options/Settings, and Change Player Profile (if applicable) core functionality available from Main Menu. 

▢ If required, Big Fish house ads enabled and Main Menu has room for them to appear (iPad - 768x75, iPhone 4 and up - 640x100). 

▢ Support button appears. 

▢ When the user taps the Support button, an integrated browser appears that points to a localized help article at BigFishGames.com. Use function [bfgManager showSupport]. 

▢ Privacy Policy button appears. 

▢ When the user taps the Privacy Policy button, an integrated browser appears that points to a localized help article at BigFishGames.com. Use function [bfgManager showPrivacy]. 

▢ Terms of Use button appears. 

▢ When the user taps the Terms of Use button, an integrated browser appears that points to a localized help article at BigFishGames.com. Use function [bfgManager showTerms]. 

▢ Big Fish Copyright and Version # appear. 

▢ All volume sliders/selectors set to 70% by default per Apple’s requirements. 

Branding

▢ Most current Big Fish branding used (included with version 3.1 and later of the SDK in bfg_gamespecific_resources). 

▢ Developer branding assets added to bfgbranding_resources.zip and included with bfg_gamespecific_resources, and branding.json is updated. 

▢ [bfgManager startBranding] function is implemented. 

In-Game Rating Prompts

If required. Contact your Big Fish Producer.

▢ Updated bfg_default_settings.json in bfg_gamespecific_resources with ratings settings. 

▢ When the user taps "Remind me later", rating prompts appear after an additional 1/4 of game play has elapsed. 

▢ When the user taps "Rate GAME TITLE now", the player is taken to the App Store page to rate the game, and rating prompts no longer appear. 

In-App Purchase

▢ In-App Purchase functional. Intialize bfgPurchase class and use IDs provided by your Producer. 

▢ In-App Purchase tested successfully by using “dummy app”; see Testing In-App Purchases for instructions.

▢ After In-App Purchase tested, Xcode project reverted to all IDs provided by Big Fish Producer. 

▢ In-App Purchase guidelines followed and new purchases saved and then "finished"; see Making Free-to-Play In-App Purchases.

Reporting

▢ Game is reporting all required events defined by bfgGameReporting class. 

▢ DEBUG enabled to check for correct calls from bfgPurchase class; see Debugging with the SDK.

Loading

▢ If load time is longer than 1 second, animation or loading message (unique to the game) appears to indicate that the device has not locked up. (Developer branding on the "Loading" screen will be rejected.) 

▢ Integrity of save data checked every time the game loads. 

▢ If primary data is corrupt, data from a secondary save slot is automatically loaded. 

▢ Required Big Fish iOS SDK components appear (localized Big Fish splash). 

Saving

▢ Game progress is stored continuously. 

▢ Game progress is backed up in secondary save slot in case of corruption. 

▢ Game saved to /Documents or using key-value storage per Apple guidelines. 

▢ Save file names provided to your Big Fish Producer. 

▢ In-App Purchases are saved immediately after purchase success notification. 

Springboard

▢ Shine removed from icon. (Final icon will be provided by Big Fish during QA.) 

▢ Bundle Display Name provided by Producer is used. Title under icon does not appear concatenated. 

Audio

▢ Turning off the ringer or using the mute button turns off all game audio. 

▢ Audio played by multi-tasking apps (e.g., iPod or Pandora) takes precedence, and game music does not play over outside audio. 

Marketing

▢ Marketing assets in .zip file returned (use the folder structure in the empty .zip file provided by your Big Fish Mobile Producer) and editable language files delivered (.xml or .txt preferred) to designated folder of EXTSVN repository. 

▢ Game-specific details provided for marketing texts: Description, number of levels/scenes, number of puzzles/mini-games. 

Xcode

▢ Build environment (Xcode version, compilers, etc.) provided to Producer (Xcode 4.5 or up). 

▢ Wildcard option set in Xcode for the “iPhone Developer” default Code Signing Identity. 

▢ Armv7 code added to .plist for iPhone target. 

▢ Custom URL scheme key added to info.plist for all targets. 

▢ Front facing camera restriction added to iPhone info.plist file to exclude 3G devices (iPhone 4 and iPod 4th gen and up, only approved devices). 

▢ Installs on all targeted devices. If any devices are excluded (for example, 3G devices or iPad 1), the game does not install. 

▢ Bundle IDs, Bundle Name, Bundle Display Name, and Project Name all follow Big Fish guidelines. 

▢ Initial source committed to EXTSVN repository at Pre-Alpha once the Xcode requirements are met. Big Fish Producer confirmed successful set up in automation system. 

Build Submission (After Beta Approval)

▢ Build committed to /trunk of assigned EXTSVN repository. 

▢ JIRA database updated with all fixes prior to every new submission. 

▢ Build notes and Revision # provided to mobile-builds@bigfishgames.com email alias in the correct email format; see Build notes.

Localization

▢ Game pre-approved for Russian only: Russian build for PC provided to Big Fish at least two months before iOS target Beta date. (Contact your Big Fish Producer if Big Fish is providing a Russian version of Lockit and it has not yet been received.) 

▢ Game pre-approved for localization only: Russian and all PC languages completed and "Ready" or "Launched" before mobile delta/diff file sent for localization. 

▢ Delta strings (those different from PC/Mac translations) added to .txt document for localization. 

▢ All text referring to "clicks, clicking, or clicked" flagged and replaced with "tap, tapping, or tapped" and added to the .txt document for localization. 

▢ All text that refers to "mouse," "cursor," or any other PC/Mac specific feature removed or replaced and added to the .txt document for localization. 

▢ If available in game, Strategy Guide checked to match iOS content, modified strings added to the .txt document for localization. 

▢ Approved marketing text and bullet points for iOS game added to the .txt document for localization. 

▢ Strings for localization in approved .txt format provided to Big Fish Producer at least two weeks before iOS target Beta date; see Localization.

▢ Translated strings merged with original PC/Mac strings and included in build by Beta. 

▢ Separate .lproj files for each supported language added to Xcode proj and reporting. 

▢ Checked for language support when iDevices are set to supported languages. 

Beta Readiness Checks

▢ Beta build is working, installs on all approved devices, and can be played all the way through. 

▢ All Loc strings delivered to Big Fish Producer in an editable format (.xml or .txt preferred). 

▢ No local copies of the Big Fish iOS SDK included with game source. Note: This does not include bfg_gamespecific_resources that need to be included locally. 

▢ Flurry events, session events (Big Fish server), revenue events (Big Fish server), and Product ID (Big Fish server) all tracking. Use DEBUG to check calls are being made correctly; see Debugging with the SDK.

▢ In-App Purchase tested successfully by using “dummy app”; see Testing In-App Purchases for instructions.
