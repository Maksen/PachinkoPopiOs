Customizing images and strings in the game specific resources.

Game Specific Resources Directory

The bfglib_gamespecific_resources directory provides an example of resources specific to each game, including image assets, game strings, and game settings. Developers must include the resources for all supported devices by replacing the example resources with the resources for their game.

To customize the resources in the game specific resources directory:

  1. SVN export the entire directory to the resources directory of your project.
  2. Replace the resources with the resources used by your game, but keep the same file names unless the description for the file specifically indicates that you can use other file names.
Note
Code in the Big Fish components refers to these file names. If you change the file names, your game will not be able to load its resources.

General Resources Requirements

Use the following standards for resources in the game-specific resources directory:

  • Use .jpg, .png, or .webp for the image format.
  • Do not compress the .zip files that contain the image bundles.
  • When you edit bfgstring files for your games, ensure that your text editor is UTF-8 compatible. If your text editor is not UTF-8 compatible, the resources will not be displayed.
  • Use the following standards for resource file names:
    • Use the format assetname-[landscape]-device-format
    • assetname describes the asset.
    • landscape indicates that the image is intended for display in landscape orientation. By default, images display in portrait orientation.
    • device indicates the type of device that can use the image. Device types include ipad, iphone, and iphone4. For iPhone-only games, the file name does not have to contain the device type.
    • format indicates the image file type, either .jpg, .png, or webp.

The following table gives examples of file names for a hypothetical “hello world“ image:

File name Supported device
helloworld.png iPhone(portrait)
helloworld-iphone.png iPhone(portrait)
helloworld-iphone4.jpg iPhone 4(portrait)
helloworld-ipad.jpg iPad(portrait)
helloworld-landscape.png iPhone(landscape)
helloworld-landscape-iphone.png iPhone(landscape)
helloworld-landscape-iphone4.jpg iPhone 4(landscape)
helloworld-landscape-ipad.jpg iPad(landscape)

Default settings

The bfg_default_settings.json file contains the default settings for Big Fish components. Typically, you use these settings without modifying them. Big Fish might request changes to the following settings:

Setting Description
allowed_ad_types An array that contains the types of ads that can be displayed. Examples of ad types include apple and bfg.
hbi Hashed bundle ID. A unique value that will be supplied by your Big Fish Producer.
dashboard_active A Boolean that determines whether the Game Discovery Network (GDN) appears. The value of this setting defaults to 1 so that the GDN dashboard appears.
dashboard_display_type The dashboard display setting. Set this value to free for Freemium games, and to paid for full games.
dashboard_show_after_this_many_runs The number of times the game must be launched before the dashboard appears. The value of this setting defaults to 0.
dashboard_timeout The number of seconds after the user leaves the game before the GDN appears again. Set the timeout to 30 during development for test purposes. When the game enters the release build stage, delete this setting to return to the default timeout value.
iap_default_product_id The Product ID for your game's default In-App Purchase. If you use the bfgPurchase APIs that do not take a Product ID, this is the ID that will be used.
flurryapikey The Flurry API key. The value of this setting defaults to “00000000000000000000“.
rating_days_until_prompt The number of days until the game displays a review prompt. In most cases, use 0 days for Freemium games, and 1 or 2 days for full games. Check with your Big Fish Producer to determine this setting. Set the value to 0 to turn off this prompt.
rating_uses_until_prompt The number of times the user must enter and leave the game before it displays a review prompt. You can use this setting in combination with the previous setting, or instead of that setting. Set the value to 0 to turn off this prompt.
rating_significant_events_until_prompt The number of significant events that must occur before the game displays a review prompt. Significant events vary by game, but often occur when a new level or scene begins. Set the value to 0 to turn off this prompt.
rating_days_before_reminding The number of days until reminding the user to rate again. Typically set to 1.

Free-to-Play Kontagent API keys

If your Free-to-Play game employs Kontagent, you should add the following code to the bfg_default_setting.json file to support an initial offline scenario:

"kontagent": {
"api_key1":"cd639bf049544df2ad6d569532dce6be",
"api_key2":"eed39bf0495664f2ad6d561222dce6be",
"start_datetime":"2013-01-01 00:00:00 PDT"
},

You will need to get the API key or API keys for your game from your Big Fish Producer.

Note
  • api_key2 is optional.
  • start_date needs to be anything valid and formatted as above.

All of these settings will be overwritten when the server is contacted.

Warning
  • Do not add these keys if you have not been told that your game supports Kontagent.

Supporting placements

If your game supports placements via PlayHaven, your Big Fish Producer will set up the PlayHaven token and secret via the Big Fish Mobile Admin tool. This data will be served up to the client via the Startup call on initial launch of the application, provided the app is online.

Once your game's settings are configured correctly, your application will receive notification if placements in your game trigger webviews to be displayed. For example, a webview may display when you call bfgGameReporting to log events, or at other SDK tracked moments in your game, such as launch, resume, etc.

Note
  • The developer's responsibility just involves integrating the Big Fish SDK and listening for notifications.
  • To add optional custom placements, use the bfgGameReporting logCustomPlacement API.
  • Custom placements are not required for Freemium games and are generally only included in Free-to-Play games.

For further information, contact your Big Fish Producer.

The following list describes potential placement notifications.

  • game_launched: Initial launch of game.
  • game_resumed: Game has resumed.
  • mainmenu_shown: Main Menu shown.
  • level_started: Level started.
  • level_finished: Level finished.
  • mainmenu_paywall_opened: Paywall opened event as a result of the user selecting the Unlock Now button on the Main Menu.
  • mainmenu_paywall_closed: Paywall closed event as a result of the user selecting the Unlock Now button on the Main Menu.
  • end_paywall_opened: Paywall open event as a result of the user reaching the end of the free trial period.
  • end_paywall_closed: Paywall close event as a result of the user reaching the end of the free trial period.
  • minigame_started: Mini game was started.
  • minigame_skipped: Mini game was skipped.
  • minigame_finished: Mini game was finished.
  • options_shown: Options was shown.
  • moregames_opened: BFG SDK Moregames UI was opened.
  • moregames_closed: BFG SDK Moregames UI was closed.
  • game_hint: A game hint was requested.
  • rate_game: The game is about to be rated.
  • achievement_earned: An achievement has been earned.
  • game_completed: The full game, including bonus chapter(s), has been completed.

Ad resources

The bfgads_bundlepackage.zip file is the current default version of the ad package as of the release date of this reference document. Contact your Big Fish Producer to get the latest version for your game.

Branding resources

The bfgbranding_resources.zip file contains branding resources.

Note
This file stores resources without compression. When you construct this file for your games, use the store or no compression setting.

The following command packages the files in the current directory into a .zip but does not compress the file:

zip -0 bfgbranding_resources.zip *

The resources in this file include a .json file (branding.json) that the bfgBranding class uses to set the orientation and display order of branding images. You must add the file names for the branding resources to branding.json. The images display in the order that they appear in the file. The bfgBranding class automatically selects the correct image to display based on the current orientation of the device.

Games for a single device type need only the images for that device and orientation. To reduce the number of images that you include, you can create and use a single square image for all orientations.

You include your own branding images instead of the images provided in the example, except for the BFGLogo image that displays for all games. Images must be saved to either .PNG, .JPG, or webP format. webP will give you the best compression. You can use whatever file names you want that meet the general resources guidelines, as long as you list the file names in branding.json.

Game Discovery Network resources

You use the following resource .zip file to customize the resources for the Game Discovery Network (GDN):

  • bfgpromodashboard_gamespecific.zip: Resources for the standard GDN UI.

Standard GDN UI resources

The bfgpromodashboard_gamespecific.zip file contains resources that you use for the Game Discovery Network, including the background (optional) and main title images, and strings for the title of your game. Replace the default images in this file with the images for your specific game, and add strings for the name of your game in one or more languages.

Note
This file stores resources without compression. When you construct this file for your games, use the store or no compression setting.

For example, the following command packages the files in the current directory into a .zip but does not compress the file:

zip -0 bfgpromodashboard_gamespecific.zip *


Resource Description
maintitle*.png The game icon. Your Big Fish Producer will provide the icons for your game for all supported devices. While testing your game, you can use the default images until Big Fish provides the specific images for your game. The screen resolution for the game icon depends upon the type of device: iPad - 132x132, iPhone - 114x114.
bfgpromodashboard_maintitle.bfgstrings This file contains the name of your game, possibly in multiple languages. The name of the game may need to be changed to support the size limitations of the Game Discovery Network. Be sure to use a UTF-8 compatible text editor when editing this file.
background*-ipad.png This is an optional resource. Typically your application's default.png is used for this screen, but you can override that by providing this image. This is the blue background for the iPad version of the Game Discovery Network. This image displays in full screen resolution and orientation.
Note
When you edit this string file for your games, ensure that your text editor is UTF-8 compatible. If your text editor is not UTF-8 compatible, the resources will not be displayed.

iPhone GDN background

When the GDN or More Games views are displayed in windowed mode, part of the background is visible to the user. On the iPad, the background is whatever the game was displaying prior to presenting the windowed view. This is usually either an in-game rendering or game menu. On the iPhone, however, the background is the default image found in the application's main bundle. The application will first look for an orientation-specific default image. If one is not found, default.png will be used. Below is the search order for the background image.

iPhone 4" screen

  • Default-Landscape-568h.png (When interface is in landscape mode.)
  • Default-Portrait-568h.png
  • Default-568h.png

iPhone 3.5" screen

  • Default-Landscape.png (When interface is in landscape mode.)
  • Default-Portrait.png
  • Default.png

iSplash! Newsletter button resources

The iSplash! resources are now generated on the fly. UIImage objects can be requested for the Main Menu button with +[bfgSplash splashMenuImageNormal] and +[bfgSplash splashMenuImageHighlightedM]. Alternatively, a fully functional UIButton that is configured to send the email and hide itself is available via +[bfgSplash createSplashButton].

PlayHaven Push Notifications

Important: Before implementing push notifications, contact your Big Fish Producer for support.

If your game employs PlayHaven push notifications, you should add the following code to the bfg_default_setting.json file. The token and secret are available from your Big Fish Producer.

"playhaven": {
"token":"4f49854752994e8fad03845149d1489e",
"secret":"d536de634b6742b495fd440931d6ffcc"
},

You can send PlayHaven push notifications to different segments of players. You can define which users will receive specific notifications (e.g., "Congrats You've won 100 gold coins") with your Big Fish Producer, who will define the receiving audience through the PlayHaven Dashboard, an administrative tool.

Besides the content, the notification also contains an icon badge number that displays on the application icon when the notification is received.

Note
There is 160-character content limit to push notifications.

Your producer can also help you send push notifications about content units, such as rewards, virtual good promotions, and announcements.

For more information, also see Integrating PlayHaven push notifications.

COPPA Compliance

To comply with COPPA regulations and standards, call the [bfgManager setCoppaOptOut] API to indicate that the user is under 13. This is set to NO by default, indicating the user is 13 or over. If the call returns YES, the user is under 13.
