Testing In-App Purchases

Testing In-App Purchases with a dummy app.

Note
While this may seem like a long procedure, it is necessary in order to verify the In-App Purchase functionality.
Important: Test any In-App Purchase functionality in the Apple sandbox prior to submitting builds to Big Fish Games.

To test the In-App Purchase system, you will need to create a dummy app using your Apple developer account, by following these instructions.

  1. Log in to iTunes Connect: https://itunesconnect.apple.com.
  2. Select Manage Your Applications.
  3. Click Add New App, and then select iOS App.
  4. Set up your basic app information with this required information:
    • Name your app anything, for example, PROJECTNAME IAP Test. The test app name MUST NOT be the same as the actual name of your game.
    • Define any SKU.
    • Define a Bundle ID, including the Bundle ID Suffix. Important: Make sure your provisioning works with the Bundle ID.
  5. Define pricing/availability. Because this is a test app, set it for the future and make it free.
  6. Define versioning information, rating, metadata, and app review information. Although this app will expire, these are still necessary settings.
  7. Upload some placeholder icons/screenshots. Big Fish can provide placeholder assets. If you need placeholder assets, ask your Big Fish Producer for them.
  8. After successfully setting up your app, the following App Information screen will display. Click Manage In-App Purchases.
  9. On the In-App Purchases page for your app, click Create New.
  10. Select the Non-consumable purchase type (only to be purchased once).
  11. Fill out the In-App Purchase Details page and click Save.
    • Pick any reference name.
    • Define a product ID. This should start with your Bundle ID followed by any unique identifier (for example, com.companyname.gamename.fullgameunlock).
    • Add at least one language.
    • Select any price tier and clear it for sale.
    • The screenshot is optional. Your dummy app will work without it.
  12. REVIEW - The most important things you just did were:
    • Associate an app to a Bundle ID that works with your provisioning file – note your Bundle ID.
    • Create an IAP Product ID for the app with Bundle ID – note your Product ID.
  13. Rebuild the project in Xcode, following these directions:
    • It must be a device build with the release scheme set.
    • Replace the Big Fish Bundle ID with the one you just set up.
    • Replace the Big Fish IAP Product ID with the one you just set up wherever it is used in the code.
    • Sign the build with your provisioning file and install it on your device.
    • Note: Several aspects of the SDK may break. This dummy build is only good for testing In-App Purchasing and game play.
  14. Test In-App Purchasing for your app.
  15. Once testing and/or bug fixing is complete, revert all the IDs back to the Big Fish supplied data, make sure your project follows Big Fish's Xcode standards, and commit to SVN.
