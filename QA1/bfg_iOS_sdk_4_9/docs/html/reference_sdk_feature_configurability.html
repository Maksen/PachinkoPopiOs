bfgGameReporting Game Event Reporting: logMainMenuShown, logRateMainMenuCanceled, logOptionsShown, logPurchaseMainMenuShown, logPurchaseMainMenuClosed, logPurchasePayWallShown, logPurchasePayWallClosed, logLevelStart, logLevelFinished, logMiniGameStart, logMiniGameSkipped, logMiniGameFinished, logAchievementEarned, logGameCompleted, logGameHintRequested, logCustomEvent, logCustomPlacement, preloadCustomPlacement Yes Optional API Wraps session end, Kontagent and PlayHaven reporting.
bfgPurchase In-App Purchase support: server validation, Apple IAP Wrapper and Purchase Reporting startService, acquireProductInformationForProducts, canStartPurchase, startPurchase, finishPurchase, restorePurchases Yes / No. It is configurable, as not all games will have IAP. But, you MUST use bfgPurchase if you have IAP in your game. You must use bfgPurchase to implement In-App Purchase in your game. ...
bfgPushNotificationManager Push notifications Yes Optional API Before implementing push notifications, contact your Big Fish Producer.
